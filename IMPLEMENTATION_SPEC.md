# SQLite Bindings for Odin — Implementation Spec

This document is the execution plan for building SQLite bindings in Odin.

The intended implementation strategy is:

1. Generate a **raw binding layer** from `sqlite3.h` using `odin-c-bindgen`
2. Keep the generated layer as close to the C API as possible
3. Build a **small handwritten wrapper layer** on top
4. Prioritize correctness, lifetime safety, and common server-side usage
5. Defer reflection-heavy mapping, ORM-like behavior, and advanced abstractions

The target runtime is a **systemic multiplayer game server** with roughly **200–300 players per server**, so the wrapper should optimize for:

- predictable statement reuse
- explicit transactions
- low allocation pressure
- solid error reporting
- safe handling of lock contention
- simple, typed APIs for common SQL usage

---

# Core principles

## Rules

- **Do not** handwrite the full SQLite C API manually
- **Do not** let the wrapper become an ORM
- **Do not** start with reflection-based row mapping
- **Do not** start with regex/tag-driven struct decoding
- **Do not** hide important SQLite behavior behind “magic” abstractions
- **Do** keep the raw binding layer mostly generated and mostly untouched
- **Do** add handwritten wrappers only where they improve ergonomics, safety, or correctness
- **Do** optimize first for the common path:
  - open database
  - prepare statement
  - bind parameters
  - step rows
  - read columns
  - reset/reuse statement
  - run transactions
- **Do** design around statement reuse and transaction boundaries

## Non-goals for v1

The following are explicitly out of scope for the first implementation unless they fall out naturally:

- ORM features
- migrations framework
- schema DSL
- query builder
- virtual tables
- extension loading UI
- custom collation helpers beyond thin raw exposure
- full coverage of obscure SQLite APIs on day one

## Current additive progress beyond the original v1 boundary

The project now includes an additive row-mapping layer on top of the explicit statement API.

This work was implemented after the raw/wrapper foundation was already in place and should be understood as optional convenience, not as a replacement for the explicit SQLite workflow.

Implemented additive mapping features now include:

- reflection-based row mapping from the current statement row into a struct
- struct-tag-based column remapping using `sqlite:"column_name"`
- thin struct query convenience wrappers for:
  - required single-row queries
  - optional single-row queries
  - all-rows queries returning a typed slice

These features remain intentionally constrained:

- explicit typed getters remain available and first-class
- mapping is additive, not mandatory
- mapping does not make the wrapper an ORM
- mapping does not change statement/transaction semantics
- mapping does not infer SQL shape beyond direct column-to-field assignment

# Deliverables

## Deliverable 1 — Raw generated binding

Produce a package that exposes the raw SQLite API generated from `sqlite3.h`.

### Requirements

- Use `sqlite-amalgamation-<version>.zip`
- Generate bindings from `sqlite3.h`
- Keep `sqlite3.c` available for vendoring/building if needed
- Use `odin-c-bindgen`
- Keep generated code separate from handwritten wrapper code
- Keep prefix stripping and renaming minimal and sane
- Keep the raw layer as faithful to the C API as possible

### Expected output

A package similar to:

- `sqlite/raw/generated.odin`
- `sqlite/raw/imports.odin`

### Acceptance criteria

- Raw package compiles
- Core SQLite symbols are available
- Platform imports work on target platforms
- The generated layer can be regenerated without manual patching

```text
sqlite/
  raw/
    generated.odin        # generated by odin-c-bindgen
    imports.odin          # foreign import logic only
  core/
    types.odin            # DB, Stmt, Blob, Backup, Error types
    errors.odin           # error helpers and result conversion
    connection.odin       # open/close/config helpers
    statement.odin        # prepare/step/reset/finalize
    bind.odin             # typed bind helpers
    column.odin           # typed read helpers
    transaction.odin      # begin/commit/rollback/savepoint
    exec.odin             # exec/query/scalar convenience
    cache.odin            # optional statement cache
    wal.odin              # busy_timeout/checkpoint helpers
  extra/
    blob.odin             # incremental blob I/O
    backup.odin           # online backup API

  tests/
    smoke.odin
    connection.odin
    statement.odin
    bind.odin
    column.odin
